# SmartTest Generator

**SmartTest Generator** — это веб-приложение для автоматической генерации тестов на основе загруженных PDF-документов с использованием технологий искусственного интеллекта (Yandex GPT). Система предназначена для преподавателей (администраторов) и студентов, позволяя быстро создавать проверочные материалы и проходить тестирование с мгновенной обратной связью.

## Основные возможности

- **Регистрация и выбор роли** — пользователь самостоятельно выбирает роль: «Администратор» или «Студент».
- **Аутентификация** — JWT-токены с контекстом авторизации (срок действия 7 дней).
- **Загрузка PDF-документов** — автоматическое извлечение текста с помощью `pdf-parse`.
- **Генерация тестов через ИИ** — с использованием Yandex GPT (или fallback-механизм на основе ключевых фраз).
- **Публикация тестов** — администратор публикует тест, делая его доступным для студентов.
- **Прохождение тестов** — пошаговый режим с одним вопросом на экране, варианты ответов A/B/C/D, индикатор прогресса.
- **Детальный разбор результатов** — подсчёт процента правильных ответов, подсветка ошибок, указание правильного варианта.
- **История прохождений** — таблица всех попыток в личном кабинете с возможностью просмотра детального разбора.
- **Управление профилем** — редактирование имени, фамилии, смена пароля, статистика (количество тестов, средний балл).
- **Две темы оформления** — светлая («капучино») и тёмная («эспрессо») с сохранением выбора в `localStorage`.
- **Адаптивный дизайн** — корректное отображение на ПК, планшетах и мобильных устройствах.
- **Журнал аудита** — логирование всех действий пользователей в отдельную таблицу БД (ГОСТ Р ИСО/МЭК 27001-2012).

## Технологический стек

| Компонент        | Технология                                  |
| ---------------- | ------------------------------------------- |
| Фронтенд         | React + Vite + React Router + Axios         |
| Бэкенд           | Node.js + Express                           |
| База данных      | PostgreSQL                                  |
| Аутентификация   | JWT (JSON Web Token)                        |
| Генерация тестов | Yandex GPT API                              |
| Стилизация       | CSS-переменные, две темы (темная / светлая) |

## Архитектура

Система реализована в архитектуре **распределённого монолита**:

- Фронтенд и бэкенд — два независимых приложения.
- Взаимодействие через REST API (формат JSON).
- Бэкенд содержит всю бизнес-логику, работает с БД и внешними API.
- Фронтенд отвечает только за отображение интерфейса и отправку запросов.

## Ролевая модель

| Роль              | Действия                                                                     |
| ----------------- | ---------------------------------------------------------------------------- |
| **Администратор** | Загрузка PDF, генерация тестов, публикация тестов, просмотр документации     |
| **Студент**       | Просмотр доступных тестов, прохождение тестов, просмотр истории и статистики |

## Ключевые сущности БД

- `users` — пользователи (email, пароль, имя, фамилия, роль)
- `documents` — загруженные PDF-файлы (название, путь, извлечённый текст)
- `quizzes` — тесты (название, сложность, проходной балл, статус публикации)
- `questions` — вопросы теста
- `answer_options` — варианты ответов (4 варианта, один правильный)
- `quiz_attempts` — попытки прохождения (процент, статус, детальные ответы в JSONB)
- `audit_logs` — журнал аудита (действия пользователей)

## Пример использования

1. **Администратор** загружает PDF-лекцию, указывает название теста, количество вопросов (5–20), сложность (лёгкий / средний / сложный) и проходной балл. Система генерирует тест через Yandex GPT.
2. **Администратор** публикует тест.
3. **Студент** видит тест на главной странице, проходит его (отвечает на вопросы по одному), получает результат в процентах и детальный разбор ошибок.
4. **Студент** может просмотреть историю всех своих попыток в личном кабинете и при необходимости отредактировать профиль или сменить пароль.
